# RAI Risk Taxonomy

Prepublication data package for mapping 1,726 global AI L4 risk cards into the new 50-L3 RAI hierarchy.

This repository is independent of the two read-only source repositories:

- `AI_Topic_Space.github.io` — frozen global 1,726-card source
- `Physical-AI-Risk-Taxonomy` — frozen 182-card Physical AI gold taxonomy

## Non-negotiable rules

- Legacy global L1-L3 fields are provenance and post-hoc audit fields only. They are not classification labels or predictive inputs.
- Physical AI's 182 placements are locked: 169 exact source IDs and 13 explicit aliases.
- Physical AI's 360 reference/justification rows and structured 3H/Role tags are preserved in the new L4 registry.
- L4 identity is permanent and independent of placement: `RAI4-0001` through `RAI4-1726`.
- L3 path IDs are concise and ordered, for example `RAI3-P-INT-05`.
- A card with no exact fit is not forced into the nearest L3. Its placement is `needs_taxonomy_decision` with `primary_l3_id: null`.
- All non-Physical placements in v1.0.0 are provisional until human gold-set validation.

## Data layout

```text

```

Canonical JSON files must be regenerated by scripts; CSV and public bundles are review and site derivatives.

## Reproduce locally

The verified build uses Python 3.13.9 and the exact versions in `requirements-lock.txt`. BGE-M3 is loaded offline at pinned revision `5617a9f61b028005a4858fdac845db406aefb181`. Local Ollama models `gemma3:4b` and `qwen3:4b` are sensitivity checks only.

```bash
python3 scripts/build_release.py                 # clean first build
# python3 scripts/build_release.py --overwrite-draft  # deliberate draft regeneration only
python3 scripts/run_llm_review.py --model gemma3:4b
python3 scripts/run_llm_review.py --model qwen3:4b
python3 scripts/build_frontier_review_packet.py
# Collect two independent hierarchy-blind expert reviews of the packet.
python3 scripts/apply_expert_consensus.py
python3 scripts/validate_release.py
```

The build is hierarchy-blind for non-Physical classification. The two local small models receive the complete 26-L3 non-Physical codebook but do not receive BGE rankings or legacy hierarchy fields; their outputs are a sensitivity audit only. A non-Physical card becomes `algorithm_proposed` only when both independent frontier expert reviewers approve the exact BGE/rule-proposed L3. Human validation is still pending.

Published release directories are never overwritten. After human approval, create a successor release with `scripts/create_remap_release.py`; it writes a complete placement diff and one append-only migration event for every changed card. Use `--dry-run` before writing and validate the successor with `scripts/validate_release.py --release-id vX.Y.Z`.

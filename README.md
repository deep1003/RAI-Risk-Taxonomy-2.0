# RAI Risk Taxonomy

Prepublication data package for mapping 1,726 global AI L4 risk cards into the new 50-L3 RAI hierarchy.

This repository is independent of the two read-only source repositories:

- `AI_Topic_Space.github.io` — frozen global 1,726-card source
- `Physical-AI-Risk-Taxonomy` — frozen 182-card Physical AI gold taxonomy

## Non-negotiable rules

- Legacy global L1-L3 fields are provenance and post-hoc audit fields only. They are not classification labels or predictive inputs.
- Physical AI's 182 placements are locked: 169 exact source IDs and 13 explicit aliases.
- Physical AI's 360 reference/justification rows and structured 3H/Role tags are preserved in the new L4 registry.
- L4 identity is permanent and independent of placement: `RAI4-0001` through `RAI4-1726`.
- L3 path IDs are concise and ordered, for example `RAI3-P-INT-05`.
- A card with no exact fit is not forced into the nearest L3. Its placement is `needs_taxonomy_decision` with `primary_l3_id: null`.
- All non-Physical placements in v1.0.0 are provisional until human gold-set validation.

Canonical JSON files must be regenerated by scripts; CSV and public bundles are review and site derivatives.
The current Pages-to-local synchronization evidence is recorded in `reports/validation/v1.0.0/live_site_sync.json`.


Open `http://127.0.0.1:8765/`. The explorer reads the exact validated public bundle and provides L1–L3 navigation, search, L3-assignment filters, pagination, and L4 detail dialogs. Classification-stage and internal hold labels are deliberately omitted from risk-card presentation.

## Experimental classification stages

- **Stage 1 (`v1.0.0`)** is the conservative baseline: 182 Physical locks, 37 frontier-consensus proposals, and 1,507 unresolved cards.
- **Stage 2 (`stage2-v1`)** preserves Stage 1 and assigns 1,334 additional cards, leaving 173 unresolved (10.0232%). These are algorithmic proposals, not human-approved truth.
- **Stage 3 (`stage3-v1`)** preserves all 1,553 Stage 2 placements and force-matches the remaining 173 to their recorded hierarchy-blind top-1 L3, producing 100% operational coverage. Forced status and original hold reasons remain explicit.

The Stage 2 and Stage 3 working reports remain source-only. The final policy assigns all 1,726 cards to the 50 L3 nodes. Fifty-five difficult cases retain a `decision_required` marker, displayed as `HOLD`; this marker is not a separate taxonomy node. The methodology, stage-specific criteria, statistics, limitations, and vector bar charts are consolidated in the canonical English `reports/latex/rai_risk_taxonomy_technical_report_2_0_en.tex` and `reports/pdf/rai_risk_taxonomy_technical_report_2_0_en.pdf`. The Korean edition is retained as a companion translation.

## Reproduce locally

The verified build uses `/opt/anaconda3/bin/python3` (Python 3.13.9) and the exact versions in `requirements-lock.txt`. The machine-readable provenance is in `environment.json`. The default macOS `python3` is not assumed to contain the required packages. BGE-M3 is loaded offline at pinned revision `5617a9f61b028005a4858fdac845db406aefb181`. Local Ollama models `gemma3:4b` and `qwen3:4b` are sensitivity checks only. The recorded `transformers==5.2.0.dev0` came from the bundled local runtime and may not resolve from public PyPI; use the recorded runtime for byte-identical reproduction.

```bash
RAI_PYTHON=/opt/anaconda3/bin/python3
"$RAI_PYTHON" -c 'import jsonschema, numpy, sentence_transformers, torch'
"$RAI_PYTHON" scripts/build_release.py                 # clean first build
# "$RAI_PYTHON" scripts/build_release.py --overwrite-draft  # deliberate draft regeneration only
"$RAI_PYTHON" scripts/run_llm_review.py --model gemma3:4b
"$RAI_PYTHON" scripts/run_llm_review.py --model qwen3:4b
"$RAI_PYTHON" scripts/build_frontier_review_packet.py
# Collect two independent hierarchy-blind expert reviews of the packet.
"$RAI_PYTHON" scripts/apply_expert_consensus.py
"$RAI_PYTHON" scripts/validate_release.py
"$RAI_PYTHON" -m unittest discover -s tests -v
```

The build is hierarchy-blind for non-Physical classification. The two local small models receive the complete 26-L3 non-Physical codebook but do not receive BGE rankings or legacy hierarchy fields; their outputs are a sensitivity audit only. A non-Physical card becomes `algorithm_proposed` only when both independent frontier expert reviewers approve the exact BGE/rule-proposed L3. Human validation is still pending.

Published release directories are never overwritten. After human approval, create a successor release with `scripts/create_remap_release.py`; it writes a complete placement diff and one append-only migration event for every changed card. Use `--dry-run` before writing and validate the successor with `scripts/validate_release.py --release-id vX.Y.Z`.
